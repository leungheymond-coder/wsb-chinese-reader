# WSB Chinese Reader — Claude Code Project Briefing

## Project Overview
A full-stack web app that automatically fetches **Wall Street Breakfast** articles from Seeking Alpha, translates them into **Traditional Chinese (繁體中文, HK style)**, stores them in a SQLite database, and emails the user when new articles are ready.

Built with Node.js + Express backend, React + Vite frontend, shadcn/ui components.

---

## 🎯 Core Requirements

### 1. Accuracy — TOP PRIORITY
- **Strict no-hallucination rule**: Only translate content actually fetched from Seeking Alpha. Never invent, embellish, or summarize beyond the source.
- Company names, tickers, and numbers MUST stay in English (e.g. Oracle (ORCL), $17.19B, +22%).
- If article content cannot be fully fetched, mark it as `incomplete` — do NOT fill gaps with assumed content.
- Translation must be paragraph-by-paragraph faithful to the source. No paraphrasing, no condensing.

### 2. Content Fetching
- Source: **Seeking Alpha WSB RSS feed** — `https://seekingalpha.com/feed/tag/wall-street-breakfast`
- **All articles** from the feed are captured — named series AND standalone pieces (e.g. "Is The Private Credit Party Over?")
- SA blocks direct scraping (PerimeterX). Fetch pipeline:
  1. **Playwright** headless browser (primary, for logged-in session)
  2. **Tavily API** (`fetchArticleViaTavily`) — primary fallback; searches SA content from its index
  3. **Gemini API** (`fetchArticleViaGemini`, model: `gemini-2.0-flash`) — secondary fallback with `googleSearchRetrieval`
  4. Claude is NOT used for article fetching — it refuses to reproduce copyrighted content
- Fetch window: **last 2 days** (EST) on each run — catches articles missed during downtime
- Store raw English source (`full_content_en`) alongside translation for accuracy verification

### 3. Scheduled Fetching
- **7:30 AM EST** daily — catches Wall Street Breakfast (published ~6–8 AM EST)
- **1:30 PM EST** daily — catches Wall Street Lunch / Roundup (published ~11 AM–1 PM EST)
- Timezone: `America/Toronto` (EST/EDT)
- Pipeline per run: fetch new/incomplete → translate pending → send email notifications
- Already-complete articles are skipped (no duplicate processing, no extra API cost)

### 4. Article Types by Day
| Day | Articles |
|-----|---------|
| Mon–Thu | Wall Street Breakfast + Wall Street Lunch |
| Friday | Wall Street Breakfast + Lunch **or** Wall Street Roundup |
| Saturday | What Moved Markets This Week |
| Sunday | Wall Street Brunch + Wall Street Week Ahead |
| Any day | Standalone articles (no series prefix) |

### 5. Database / Storage
- **SQLite** via Node 24 built-in `node:sqlite` (`DatabaseSync`) — no native compilation needed
- Schema (articles table):
  ```
  id, sa_url, sa_article_id, title_en, title_zh,
  summary_en, summary_zh, key_points_en (JSON), key_points_zh (JSON),
  full_content_en, full_content_zh,
  catalyst_watch_en, catalyst_watch_zh,
  tickers (JSON), thumbnail_url,
  published_at, fetched_at, translated_at,
  fetch_status (pending | complete | incomplete | failed),
  translate_status (pending | complete | failed),
  email_sent (boolean)
  ```

### 6. Translation
- **Claude API** (`claude-sonnet-4-5`) for all translation
- Full HK Traditional Chinese rules — see `/Users/heymondleung/.claude/skills/wsb-translator.md`
- Key rules:
  - Traditional Chinese (HK) ONLY — never Simplified
  - Company names, tickers, people names stay in English
  - HK terminology: 聯儲局 (not 美联储), 業績勝預期, 盤前, 按年, etc.
  - Paragraph-by-paragraph faithful translation
- **Program title mapping** (series articles get a prefix):
  | Series | Chinese Title |
  |--------|--------------|
  | Wall Street Breakfast | 華爾街早市快訊 |
  | Wall Street Lunch | 華爾街午間財經 |
  | Wall Street Brunch | 華爾街投資動向 |
  | Wall Street Roundup / RoundUp | 華爾街收市總結 |
  | What Moved Markets This Week | 本週市場焦點 |
  | Wall Street Week Ahead | 華爾街下週前瞻 |
- **Title format**: `華爾街早市快訊：Oracle 業績勝預期 股價急升` (series) or plain translated title (standalone)
- No tickers in translated titles
- Key points: generated by Claude if missing or detected as podcast chapter markers

### 7. Email Notifications
- Send email when new articles are translated and ready
- Nodemailer with Gmail SMTP
- Subject: `【華爾街早報】新文章已翻譯 — [Article Title in Chinese]`
- Body: article title (ZH + EN), summary, key points in Chinese, link to web app
- Multiple articles batched into one email per run

### 8. Web App (Frontend)
- React + Vite, served as static files by Express
- shadcn/ui components (Button, Alert, Toaster/Sonner, ToggleGroup)
- Seeking Alpha dark red/gold theme with CSS variables
- Mobile-first responsive design
- **Listing page**: cards with thumbnail, date (EST), Chinese title, EN subtitle, key point bullets
- **Detail page**: full article with EN/中文繁體 language toggle
- Dates displayed in EST (`America/Toronto` timezone)

## ⚠️ Known Issues & Solutions

- **SA PerimeterX blocks Playwright**: SA denies headless browser access. Primary fallback is Tavily (has SA content cached in its search index). Secondary is Gemini with `googleSearchRetrieval`.
- **Tavily returns markdown with URLs**: `[ADBE](https://...)` syntax pollutes content. `cleanMarkdown()` in fetcher strips all markdown link syntax.
- **Podcast chapter markers as key points**: Detected by `hasGoodKeyPoints()` (checks for `(0:25)` timestamp patterns). Claude regenerates clean key points via `generateKeyPoints()`.
- **Rate limits**: Translation calls are sequential with 300–500ms delays. Never concurrent.
- **Older articles**: Articles more than ~2 days old may not be in Tavily's index and cannot be recovered.

# Manual fetch + translate + email pipeline
curl -X POST http://localhost:3000/api/fetch

# Run translator only (re-translate pending articles)
node backend/translator.js

# Run scheduler standalone (with immediate trigger)
node backend/scheduler.js --run-now

# Rebuild frontend after UI changes
cd frontend && npm run build
