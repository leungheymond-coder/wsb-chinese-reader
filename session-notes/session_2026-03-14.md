# Session Wrap-Up — 2026-03-14

## What We Built / Changed

### Scheduler — Sleep Resilience
- Added `recoverMissedExecutions: true` to both cron jobs (node-cron recovers missed fires after Mac wakes from sleep)
- Changed schedule from fixed times (`30 7 * * *`, `30 13 * * *`) to rolling 30-min windows:
  - Morning: `*/30 7-10 * * *` (every 30 min, 7–10 AM EST)
  - Afternoon: `*/30 12-15 * * *` (every 30 min, 12–3 PM EST)
- Mac now only needs to be awake for any 30-min slot in those windows

### macOS pmset Wake Schedule
- Set `sudo pmset repeat wake MTWRFSU 07:25:00` — Mac wakes from sleep at 7:25 AM daily
- Requires charger plugged in for reliable wake
- 5-minute buffer before first 7:30 AM cron tick

### Scheduler — Always Notify
- `notifyPending()` now called unconditionally at end of every pipeline run (previously only called if `fetched > 0`)
- Prevents missed emails when articles were translated in a prior run but notification failed

### Gemini Fallback — Model + API Update
- Updated model: `gemini-2.0-flash` → `gemini-2.5-flash` (old model was returning HTTP 404)
- Updated search tool: `googleSearchRetrieval` → `google_search` (API format changed in Gemini 2.5)

### Fetcher — Content Parser Fallback
- Added "Fallback 2" to `parseWSBArticle()`: when no "Summary" or "This is an abridged transcript" markers exist (e.g. weekly summary articles like "What Moved Markets This Week"), use all body text as content
- Previously these articles returned empty `full_content_en` even when raw text was successfully fetched

### Fetcher — Incomplete Content Detection
- Fetcher now marks articles as `incomplete` (not `complete`) when parsed `full_content_en` is under 100 chars, even if raw page text was long enough to pass the paywall check
- Prevents articles being silently stuck with empty content + `fetch_status=complete`

### Fetcher — Tavily Thumbnail
- `fetchArticleViaTavily()` now returns `thumbnail_url: match.image || ''`
- Tavily API results include an `image` field — we were discarding it previously
